Introduction

What SkyKoi is and how it deploys AI agents for your business.

SkyKoi is a B2B platform that deploys and manages dedicated AI agent instances for businesses. Each customer gets a dedicated EC2 instance running an isolated AI assistant powered by Claude Opus 4.6 via AWS Bedrock — with full shell access, file operations, web browsing, and more.

What is SkyKoi?

Running a dedicated AI agent requires provisioning a server, installing dependencies, configuring the Gateway daemon, setting up TLS, managing IAM roles, and wiring up channel credentials. That's a lot of infrastructure work before you can even start chatting.

SkyKoi abstracts all of that complexity into a single experience. You sign up, visit the Chat page, and a dedicated EC2 instance is automatically provisioned for you — complete with the Koi runtime, nginx, TLS, and a live AI agent. The entire provisioning process takes about 2–3 minutes.

Key Features

  • •Dedicated EC2 Instance — Each user gets their own t3.small instance (Ubuntu 22.04) in us-east-1 with complete process, filesystem, and network isolation
  • •Auto-Provisioning — On first visit to Chat, your server is launched and configured automatically — no manual setup required
  • •Claude Opus 4.6 via AWS Bedrock — Powered by amazon-bedrock/us.anthropic.claude-opus-4-6-v1 for enterprise-grade AI capabilities
  • •Full Agent Runtime — Your agent can read/write files, execute shell commands, browse the web, search the internet, analyze images, generate speech, run cron jobs, and spawn sub-agents
  • •Infrastructure as Conversation — Agents can manage AWS resources (EC2, S3, Lambda, RDS, CloudFormation) through natural language
  • •WebChat Interface — Real-time chat via WSS (nginx reverse proxy with self-signed TLS)
  • •Team Collaboration — Role-based access control with Owner, Admin, Member, and Viewer roles
  • •Usage Analytics — Real-time dashboards showing token usage, conversation metrics, and agent health
  • •Enterprise Security — Clerk auth, token-based Gateway auth, dedicated instance isolation, and IAM roles

The Gateway is a long-lived daemon that runs as a systemd service. The embedded agent runtime uses Claude Opus 4.6 via AWS Bedrock with workspace files (AGENTS.md, SOUL.md, USER.md, TOOLS.md, IDENTITY.md), persistent sessions stored as JSONL, and a full tool system.

💡Everything described in these docs reflects real, working infrastructure — not vaporware. SkyKoi is a production platform built on proven agent technology.
